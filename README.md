
# Student-Teacher Feature Pyramid Matching for Anomaly Detection (STPM)

## Project Overview
This project implements an unsupervised anomaly detection system based on the Student-Teacher Feature Pyramid Matching (STPM) framework. The implementation is specifically calibrated for the 'Metal Nut' category of the MVTec Anomaly Detection (MVTec AD) dataset. The goal is to detect and localize manufacturing defects using only defect-free images for training.

## System Architecture
The architecture consists of two main components: a pre-trained Teacher network and a trainable Student network.

1. Teacher Network: A ResNet-18 backbone pre-trained on ImageNet. Its weights are frozen and serve as the reference for 'normal' feature distributions.
2. Student Network: Identical in architecture to the Teacher network. It is initialized with ImageNet weights but is trained to minimize the distance between its feature maps and the Teacher's feature maps on normal samples.
3. Feature Pyramid: Features are extracted from three distinct layers (conv2_x, conv3_x, and conv4_x) to capture multi-scale information, from low-level textures to high-level semantic structures.

## Mathematical Formulation
The anomaly detection is driven by the Normalized Euclidean Distance between the feature maps.

Loss Function:
L = 0.5 * || F_teacher_norm - F_student_norm ||^2

At inference, an anomaly map is generated by calculating the pixel-wise difference between the Teacher and Student features across the pyramid levels, followed by bilinear interpolation to match the original image resolution.

## Configuration and Hyperparameters
- Input Resolution: 256x256 pixels
- Backbone: ResNet-18
- Optimizer: Stochastic Gradient Descent (SGD)
- Learning Rate: 0.4 (with MultiStepLR decay at 100 and 200 epochs)
- Momentum: 0.9
- Weight Decay: 1e-4
- Training Epochs: 300
- Batch Size: 32

## Performance Metrics
The model achieved the following results on the Metal Nut test set:
- AUC-ROC: ~0.8431
- Classification Threshold: 0.055
- Performance Note: Metal Nut is considered a challenging category in the MVTec dataset due to the subtle nature of defects like scratches or slight deformations.

## Installation and Usage
Dependencies include PyTorch, Torchvision, OpenCV, Matplotlib, and Scikit-learn.

To run the project in a Colab environment:
1. Mount Google Drive to access the dataset and save weights.
2. Configure the directory paths in the Project Configuration cell.
3. Execute the training loop or load the provided pre-trained weights.
4. Use the inference functions to generate anomaly heatmaps and classification verdicts.

## Directory Structure
- weights/: Stores the best performing model (stpm_best.pth).
- results/heatmaps/: Contains generated visualizations and ROC curves.
- STPM_Project.ipynb: The primary execution notebook.
